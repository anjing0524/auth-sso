## 变更概览

| # | 决策 | 影响范围 |
|---|------|----------|
| ADR-006 | JWT 最小化（移除 roles/permissions/deptIds） | JWT Claims 定义、Token 签发/验签、续签逻辑 |
| ADR-006 | `aud="auth-sso"`、`iss="auth-sso"`（体系级标识） | 同上 |

| ADR-007 | 子应用自取 Redis 权限 | Portal `checkPermission/withAuth/withPermission` |
| ADR-008 | 权限码 `{clientId}:{resource}:{action}` 命名空间化 | permissions 表、seed、contracts |
| ADR-008 | 删除 `resource`/`action` 列 | permissions 表 |
| ADR-008 | 保留 `client_id` 冗余列 | permissions 表 |
| ADR-006 | Refresh Token 去 `clientId`，纯用户级 | refresh_tokens 表、签发/轮换/撤销函数 |

## Phase 1: Schema & Migrations

### 1.1 permissions 表变更

**文件**: `apps/portal/src/db/schema/rbac.ts`

| 操作 | 内容 |
|------|------|
| 删除列 | `resource` varchar(100) |
| 删除列 | `action` varchar(50) |
| 扩展列 | `code` varchar(50) → varchar(150) |
| 删除约束 | `resource IS NOT NULL AND action IS NOT NULL` for API type |
| 保留 | `client_id` FK（冗余索引列） |

### 1.2 refresh_tokens 表变更

**文件**: `apps/portal/src/db/schema/auth.ts`

| 操作 | 内容 |
|------|------|
| 删除列 | `client_id` varchar(50) |
| 删除索引 | `idx_refresh_tokens_client` |
| 删除索引 | `idx_refresh_tokens_user` — 改为普通索引（仅 userId） |

### 1.3 Migration SQL

```sql
-- permissions
ALTER TABLE permissions DROP COLUMN resource;
ALTER TABLE permissions DROP COLUMN action;
ALTER TABLE permissions ALTER COLUMN code TYPE varchar(150);
ALTER TABLE permissions DROP CONSTRAINT permissions_type_check;
ALTER TABLE permissions ADD CONSTRAINT permissions_type_check CHECK (
  (type IN ('DIRECTORY','PAGE') AND client_id IS NULL)
  OR (type = 'API')
);

-- refresh_tokens
ALTER TABLE refresh_tokens DROP COLUMN client_id;
DROP INDEX idx_refresh_tokens_client;
CREATE INDEX idx_refresh_tokens_user ON refresh_tokens(user_id);
```

### 1.4 Drizzle Schema 同步更新

- `apps/portal/src/db/schema/rbac.ts`: 删除 `resource`/`action` 字段定义，更新 CHECK
- `apps/portal/src/db/schema/auth.ts`: 删除 `clientId` 字段定义，删除 `idx_refresh_tokens_client`
- Drizzle relations 更新（如有）

## Phase 4: 权限上下文 Redis 化

### 4.1 getUserPermissionContext 增强

**文件**: `apps/portal/src/lib/permissions.ts`

当前已有 Redis 缓存层。需确保：
- RBAC 变更（角色分配、权限分配、部门变动）时 **主动更新** Redis `user:{sub}:perms`
- `cacheUserPermissionContext()` 在 Token 续签时预填充

### 4.2 Redis Key 格式

```
user:{sub}:perms → { roles: string[], permissions: string[], deptIds: string[] }
```

### 4.3 变更触发点

| 变更操作 | 触发 Redis 更新 |
|----------|----------------|
| 用户-角色分配/移除 | 该用户的 `user:{sub}:perms` 刷新 |
| 角色-权限分配/移除 | 该角色下所有用户的 `user:{sub}:perms` 刷新 |
| 部门新增/移动 | 该部门子树下所有角色对应用户的 deptIds 刷新 |
| Token 续签 | 续签时校验 Redis 与 DB 一致性 |

## Phase 5: Portal 自身鉴权改造

Portal 自身也是子应用，遵循 ADR-007（子应用自取鉴权）。

### 5.1 checkPermission 改造

**文件**: `apps/portal/src/lib/auth/check-permission.ts`

改造点：
- 当前从 `claims.roles`/`claims.permissions` 读取
- 改为从 Redis `user:{sub}:perms` 读取
- `claims` 不再包含 roles/permissions/deptIds，`PermissionCheckResult` 中移除 `claims` 字段或调整为最小化

```typescript
// New flow
export async function checkPermission(options: PermissionCheckOptions): Promise<PermissionCheckResult> {
  const identity = await resolveIdentity();  // returns { userId }
  if (!identity) return { authorized: false, error: '未登录', statusCode: 401 };

  const { userId } = identity;
  
  // Read from Redis
  const permsData = await redis.get(`user:${userId}:perms`);
  if (!permsData) return { authorized: false, error: '鉴权服务不可用', statusCode: 503 };
  
  const { roles, permissions } = JSON.parse(permsData);
  
  // Admin bypass
  if (roles.some(r => ADMIN_ROLE_CODES.includes(r))) {
    return { authorized: true, userId };
  }
  
  // Permission check
  if (options.permissions?.length) {
    const ok = options.requireAll
      ? options.permissions.every(p => permissions.includes(p))
      : options.permissions.some(p => permissions.includes(p));
    if (!ok) return { authorized: false, userId, error: '权限不足', statusCode: 403 };
  }
  
  return { authorized: true, userId };
}
```

### 6.3 调用方更新

| 调用位置 | 变更 |
|----------|------|
| `apps/portal/src/app/api/auth/oauth2/token/route.ts` | token 端点移除 `client_id` 传参 |
| `apps/portal/src/app/api/auth/refresh/route.ts` | 续签端点移除 `client_id` 传参 |
| `apps/portal/src/app/api/auth/logout/route.ts` | logout 已用 `revokeAllRefreshTokens(userId)`，无变化 |

### 6.4 OAuth /token 端点 Client 认证

ADR-006 决定 RT 去 clientId，但 `/token` 端点仍需 Client 认证（`client_id` + `client_secret`）：

- `authorization_code` grant：Client 认证保持不变
- `refresh_token` grant：Client 认证保持不变（client_id + client_secret 校验），但不再用于查询 RT——RT 仅用 `tokenHash` 查找，client_id 仅用于凭证校验
