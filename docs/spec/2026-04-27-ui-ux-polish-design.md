# UI/UX Polish 设计与交互升级规范

日期: 2026-04-27
基于 `final-ui-verification-report.md` 中的截图与 `DESIGN.md` 的规范，针对当前系统存在的布局层级失衡、操作流打断以及交互反馈缺失等问题，提出以下优化升级方案。

## 1. 门户工作台 (Dashboard) 优化

**问题**：空状态单薄，右侧视觉层级不当，顶部元素未对齐。

**升级方案**：
*   **安全审计日志区域**：采用数据表格的骨架屏（Skeleton）作为背景，叠加 `gray-500` 颜色的说明文本“系统最新安全审计日志将在这里显示”。
*   **快捷直达卡片**：增加悬停交互，卡片在 `:hover` 状态下背景色转为 `primary-50` (`#E6F0FF`)，并添加平滑过渡动画 (`transition-all duration-200 ease-out`)。
*   **标题栏对齐**：使用 Flexbox 确保“工作台”标题与右侧的“导出报告”按钮在同一水平线上垂直居中对齐 (`items-center`)。

## 2. 长表单操作流优化 (Sticky Footer)

**问题**：以“新增用户”页面为代表的长表单，提交按钮固定在右上角，用户填写到底部后需滚回顶部提交，打破操作连续性。

**升级方案**：
*   **吸底操作栏**：移除页面右上角的“确认创建”按钮。在表单页面底部引入一个固定在底部的操作栏（Sticky Footer）。
*   **视觉规范**：操作栏具有白色背景，顶部带有 1px 的 `gray-200` 边框，内部包含左对齐或右对齐的按钮组（“取消”为 Ghost/Secondary 样式，“确认创建”为 Primary 样式）。

## 3. 弹窗 (Modal) 遮罩优化

**问题**：“新建角色”等对话框背景使用了极高强度的毛玻璃效果，完全遮挡了底层列表数据。

**升级方案**：
*   **微模糊 + 半透明**：调整 Backdrop 的样式。移除当前的高强度模糊。
*   **样式应用**：使用 Tailwind 类似工具的 `bg-slate-900/30` (`rgba(15, 23, 42, 0.3)`) 结合 `backdrop-blur-sm` (4px 的模糊半径)，既凸显当前操作焦点，又保留底层上下文。

## 4. 微交互打磨 (Micro-interactions)

**问题**：缺乏基础的表单交互反馈（密码显隐、搜索清除、选项释义、明确的空状态引导）。

**升级方案**：
*   **密码输入框**：增加“眼睛”图标，支持 `type="password"` 与 `type="text"` 之间的点击切换。
*   **搜索输入框**：当输入框有值时，右侧显示清除（X）图标，点击后清空输入并重新获取/重置列表数据。
*   **选择器辅助文本**：为涉及业务逻辑的下拉选择器（如角色的“默认数据范围”）下方增加 `xs` 号 (`12px`)、颜色 `gray-500` 的提示文案。
*   **树形/列表空状态**：对于“菜单管理”等核心资源页面，若数据为空，不再展示模糊的“未找到匹配的菜单项”，而是居中展示空状态图形，并提供明显的 **“+ 创建第一个菜单”** 引导按钮。

## 验证与验收

*   **视觉**：所有调整均需符合 `DESIGN.md` 中定义的间距、颜色变量（如 `primary-500`, `gray-200` 等）、圆角（如表单控件 8px）以及动效（200ms ease-out）标准。
*   **交互**：表单在内容填写后滚动，操作栏应稳定吸附在视口底部；弹窗开启时能隐约看见背后的数据内容。
